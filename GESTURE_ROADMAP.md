# 手势特效开发规划 (Gesture Effects Roadmap)

本文档基于现有“爱心雨”特效的开发经验，总结了未来计划加入的互动手势及其技术实现方案。所有特效均遵循“**即时触发 - 持续保持 - 离手渐隐**”的交互逻辑。

> 状态说明（2026-06-21）：本文档保留早期方案作为设计记录。“蜘蛛侠吐丝”和“能量球”因误触率较高，已从最终演示版代码中移除；当前保留爱心、摇滚和剪刀手特效。

## 1. 蜘蛛侠吐丝 (Spider-Man Web)

模拟超级英雄发射蛛丝的经典动作，增加趣味性和角色扮演感。

- **手势描述**：
  - 手掌朝前或朝上。
  - **伸直**：拇指、食指、小指。
  - **弯曲**：中指、无名指。
- **识别逻辑 (MediaPipe)**：
  - `IndexTip(8)` & `PinkyTip(20)` & `ThumbTip(4)` 距离手掌较远（伸直状态）。
  - `MiddleTip(12)` & `RingTip(16)` 距离手掌较近（弯曲状态）。
  - 触发阈值需稍微放宽，避免手指稍微弯曲导致中断。
- **视觉特效**：
  - **触发瞬间**：从手腕 (`Wrist 0`) 位置射出一条或多条白色半透明蛛丝。
  - **持续状态**：蛛丝粘附在屏幕边缘（随机位置）或形成一张半透明的蜘蛛网覆盖在前景。
  - **消失**：手势结束后，蛛丝/蛛网快速溶解或断裂消失。

## 2. 摇滚时刻 (Rock & Roll)

营造热烈、激情的氛围，适合音乐场景或情绪表达。

- **手势描述**：
  - 经典的“金属礼”手势 (Sign of the Horns)。
  - **伸直**：食指、小指。
  - **弯曲/压住**：中指、无名指、拇指（拇指压在弯曲的手指上）。
- **识别逻辑**：
  - `IndexTip(8)` & `PinkyTip(20)` 伸直。
  - `MiddleTip(12)` & `RingTip(16)` 弯曲。
  - `ThumbTip(4)` 靠近 `Middle/Ring` 的指节。
- **视觉特效**：
  - **粒子系统**：指尖 (`IndexTip`, `PinkyTip`) 喷射出火焰粒子或电火花。
  - **屏幕滤镜**：屏幕边缘闪烁红/紫色光晕，或出现轻微的震动效果 (Shake Effect)。
  - **装饰元素**：随机浮现吉他拨片、骷髅头、闪电符号。
  - **音频 (可选)**：触发激昂的电吉他 Riff 音效。

## 3. 胜利/开心 (Victory / Peace)

通用的拍照和正向情绪表达手势。

- **手势描述**：
  - 经典的“剪刀手” (V Sign)。
  - **伸直**：食指、中指（分开呈V型）。
  - **弯曲**：拇指、无名指、小指。
- **识别逻辑**：
  - `IndexTip(8)` & `MiddleTip(12)` 伸直且有一定间距。
  - 其他手指弯曲。
- **视觉特效**：
  - **萌系路线**：在两根手指上方生成可爱的卡通兔耳朵。
  - **派对路线**：从指尖喷射出彩带 (Confetti) 和气球上升。
  - **美颜**：触发轻度美颜滤镜或贴纸效果。

## 4. 魔法聚气 (Energy Orb)

双人或双手互动的高级特效，利用双手距离控制特效参数。

- **手势描述**：
  - 双手掌心相对，呈抱球状（类似龟派气功）。
- **识别逻辑**：
  - 检测到**双手**。
  - 双手掌心 (`Wrist 0` 或 `IndexMCP 5`) 距离在一定范围内 (e.g., 100px - 400px)。
  - 所有手指自然微曲或伸直，主要判断双手相对位置。
- **视觉特效**：
  - **中心汇聚**：在双手中心点生成一个旋转的能量球（光效、闪电纹理）。
  - **交互性**：
    - 双手靠近 -> 能量球变小但颜色变深（高能压缩）。
    - 双手拉开 -> 能量球变大但颜色变浅。
  - **消失**：双手分开超过阈值或撤销手势，能量球向四周消散。

## 5. 点赞 (Thumbs Up)

直播互动或确认操作的通用手势。

- **手势描述**：
  - 竖起大拇指。
- **识别逻辑**：
  - `ThumbTip(4)` 竖直向上（y 坐标小于其他关节）。
  - 其他四指弯曲。
- **视觉特效**：
  - 拇指上方弹出一个巨大的 3D 风格 👍 图标。
  - 伴随无数小的 "+1" 或金色星星粒子向上飘动。

## 技术架构建议 (Refactoring for Scalability)

为了支持上述多种手势的扩展，建议对现有代码进行以下重构：

1.  **手势检测配置化**：
    在 `GestureDetector` 中建立一个手势配置表，而不是写死 `if-else`。
    ```javascript
    const Gestures = {
        HEART: { check: (landmarks) => ... },
        SPIDERMAN: { check: (landmarks) => ... },
        ROCK: { check: (landmarks) => ... }
    };
    ```

2.  **特效工厂模式**：
    `EffectManager` 根据检测到的手势名称，动态实例化或激活对应的 `Effect` 类。
    - `HeartEffect`
    - `SpiderManEffect`
    - `RockEffect`
    所有 Effect 类继承自 `BaseEffect`，并必须实现 `start()` (启动持续特效) 和 `stop()` (渐隐结束) 接口。

3.  **统一生命周期管理**：
    确保同一时间只有一个主特效处于激活状态（互斥），或者设计优先级系统（例如“魔法聚气”优先于“点赞”）。
